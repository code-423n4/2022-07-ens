premm.eth (nxt3d) 07/17/2022 1:02 AM
If you own a name, you can create as many subdomains as you like, if you own those subdomains, then you can make sub subdomains. If the name is wrapped in the NameWrapper then those subdomains are ERC-1155 NFTs. This is the main function of the NameWrapper. If however, you don't want to register one sub at a time, using your externally owned account (EOA), that's where a registrar comes in. A registrar is a smart contract that you give the same permissions to as your EOA, think of it like a box office for a theater, it only does what the owner wants. If you want it to create subdomains and issue them to the public, for a fee for example, then you can create a contact that does that. I think the confusion lies in the fact that .eth is actually a independent domain, so it has an owner, and that owner happens to be a registrar, which is also a wrapper of .eth subs, i.e., 2LD domains, e.g., vitalik.eth. The composable nature of ENS names, means that things can be configured in different ways, but there are base structures, e.g. wrappers, registrars, and controllers, and these things all can come together in different ways. If you tell me your application I can tell you how it could be set up. (edited)

premm.eth (nxt3d) 07/17/2022 2:24 AM
The first two fuses that need to be set on subnames in order to set other fuses are PARENT_CANNOT_CONTROL and CANNOT_UNWRAP. This effectively protects the subname owner from the parent taking the name back, as long as the expiry of the fuses of the subname is not expired. If these fuses are not set, then the parent name owner, e.g., defi.eth, can call setSubnodeOwner, or setSubnodeRecord to set the owner to himself, and then call unwrap, to fully take back and unwrap the name.
